V2X.tools
HTTP API — decode, encode, convert and generate ETSI V2X messages
Concepts Convert Generate Discover Errors

Base URL: https://v2x-tools-backend-production.up.railway.app

The API converts ETSI V2X messages between three encodings — UPER, XML and JSON — and generates sample messages. All requests are plain HTTP; no authentication is required for the public message set.

Concepts

Encodings

  • UPER — binary. Sent/received as hex (text, optional uper: prefix) or as raw bytes.
  • JSON / XML — text, human-readable.

Identifying the message (X-Ref)

UPER is not self-describing, so the API must know which message definition to use. Three ways:

  • Auto-detect — omit X-Ref. The API reads the ETSI ITS header (protocolVersion + messageId) from the payload and picks the matching message. The simplest option for decoding.
  • Message ref — X-Ref: cam_v2. A named, pre-configured message (see Discover).
  • Type ref — X-Ref: cam_v2:CAM (alias:Type). A raw ASN.1 type from a module, no header defaults.

Convert

POST /api/convert

Convert a payload from one encoding to another (decode, encode, or re-encode).

HeaderRequiredDescription
X-FromrequiredInput encoding: UPER, XML or JSON.
X-TorequiredOutput encoding.
X-RefoptionalMessage/type ref. Omit to auto-detect from the payload header.
Content-Typeoptionaltext/plain for hex/JSON/XML (default), or application/octet-stream for raw binary input.
Acceptoptionalapplication/octet-stream to receive UPER as raw bytes instead of hex.

The request body is the payload. The response body is the converted payload (text, or bytes when requested).

Decode UPER → JSON (auto-detect)

curl -X POST "https://v2x-tools-backend-production.up.railway.app/api/convert" \
  -H "X-From: UPER" -H "X-To: JSON" \
  -H "Content-Type: text/plain" \
  --data "uper:02024c1cd2893d45035379f3eb4ed18dd0..."

# → {"header":{"protocolVersion":2,"messageID":2,"stationID":...},"cam":{...}}

Encode JSON → UPER (explicit ref)

curl -X POST "https://v2x-tools-backend-production.up.railway.app/api/convert" \
  -H "X-Ref: cam_v2" -H "X-From: JSON" -H "X-To: UPER" \
  -H "Content-Type: text/plain" \
  --data '{"header":{"protocolVersion":2,"messageID":2,"stationID":1},"cam":{...}}'

# → uper:0202...

Decode raw binary

curl -X POST "https://v2x-tools-backend-production.up.railway.app/api/convert" \
  -H "X-From: UPER" -H "X-To: JSON" \
  -H "Content-Type: application/octet-stream" \
  --data-binary @message.bin
Auto-detect and the standard header read work for UPER, JSON and XML. A payload whose (protocolVersion, messageId) is not a known message returns a clear error — pass an explicit X-Ref in that case.

Generate

POST /api/generate

Generate a sample message in the requested encoding.

HeaderRequiredDescription
X-RefrequiredWhich message to generate (e.g. cam_v2, or a type ref cam_v2:CAM).

JSON body (all optional):

FieldDescription
formatOutput encoding: UPER (default), JSON, XML.
sizeHow full the random sample is: SMALLEST, SMALL (default), MEDIUM, BIG, BIGGEST.
minimaltrue = only mandatory fields with default values (ignores size).
curl -X POST "https://v2x-tools-backend-production.up.railway.app/api/generate" \
  -H "X-Ref: cam_v2" -H "Content-Type: application/json" \
  --data '{"format":"UPER","size":"SMALL"}'

# → uper:0202...

Discover

Read-only endpoints to discover the available messages and modules (the valid X-Ref values).

GET /api/messages

Lists the pre-configured messages. Each name is a valid X-Ref.

curl "https://v2x-tools-backend-production.up.railway.app/api/messages"

# → [{"name":"cam_v2","moduleAlias":"cam_v2","rootType":"CAM","description":"...","fixups":[...]}, ...]

GET /api/modules

Lists the ASN.1 modules with their OID and aliases (use alias:Type as a type ref).

curl "https://v2x-tools-backend-production.up.railway.app/api/modules"

# → [{"oid":"CAM_PDU_Descriptions {...}","moduleId":"cam_pdu_descriptions_f73e8d","aliases":["cam_v2"]}, ...]

Errors & status codes

  • 200 — success; body is the converted/generated payload (or the requested list).
  • 400 — the payload could not be decoded/encoded; body is {"error": "..."}.
  • 404 — unknown ref, or auto-detect found no message for the payload's header.
